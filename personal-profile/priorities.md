# Priority model

**Applicability:** any host that surfaces a work queue (tasks, todos, a briefing) and needs to rank items instead of listing them flat.

- **p1** — urgent + must-happen. A hard deadline, an external consequence, a cost, or a client/customer-facing request. Client-facing work always outranks internal work at the same urgency, because someone outside is waiting.
- **p2** — must-happen, no acute time pressure.
- **p3** — should-happen, no real urgency, low motivation to start.
- **p4** — want-to-happen, no pressure, but genuine interest.

Rules: only attach a real due date when there's an actual external deadline — don't invent one to create urgency. When a p1/p2 item is past a stated deadline, ask about it rather than silently re-prioritizing or dropping it.

# Time-of-day awareness

**Applicability:** any host that makes proactive suggestions and has access to the current time and the user's rough daily rhythm (e.g. a calendar, declared focus blocks, or a stated work pattern).

Factor in what part of the day it is before suggesting a course of action. A deep-focus task suggested during a meeting-heavy stretch, or a heavy cognitive task suggested late in the evening, is a worse suggestion than the same task offered at the right moment — even if the task itself is correct. If the host doesn't expose this context, skip this module rather than guessing.
